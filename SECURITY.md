# Security Policy

## Supported Versions

Forge Map 3D is pre-release. Security fixes are made on the current default
branch and included in the next release. Older snapshots are not supported.

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability or publish exploit
details before a fix is available.

Use GitHub's private vulnerability reporting option in the repository's
**Security** tab. If that option is unavailable, contact
`security@forgemap3d.com`.

Include the affected route or component, reproduction steps, impact, and any
suggested mitigation. Remove personal location data, credentials, and unrelated
third-party data from the report.

You should receive an acknowledgement within seven days. The maintainer will
coordinate validation, remediation, and disclosure with the reporter.

> Repository owner: configure forwarding for `security@forgemap3d.com` and
> enable GitHub private vulnerability reporting before public launch.

## Scope

Reports about Forge Map 3D code and its deployment configuration are in scope.
Availability or data-quality problems in OpenStreetMap, 3DBAG, PDOK, or public
Overpass instances should be reported to the relevant upstream project unless
Forge Map 3D handles them unsafely.
